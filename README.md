## Опис предметної області (Моделі даних)

У системі виділено дві основні сутності, які описують процес трекінгу здоров'я. Кожна модель (data class) має щонайменше 4 поля з використанням різних типів даних (`Int`, `String`, `Boolean`, `Float`).

### 1. `Vitamin` (Вітаміни та добавки)
Описує конкретний вітамін, який користувач має приймати.
* `id: Int` - унікальний ідентифікатор запису.
* `name: String` - назва вітаміну.
* `dailyDosageMg: Int` - рекомендована добова доза у міліграмах.
* `isEssential: Boolean` - прапорець, що вказує, чи є вітамін критично важливим для щоденного прийому.
* `category: String` - категорія.

### 2. `WaterRecord` (Запис про випиту воду)
Фіксує кожен прийом води користувачем.
* `id: Int` - унікальний ідентифікатор транзакції.
* `amountMl: Int` - об'єм випитої води у мілілітрах.
* `timeAdded: String` - час додавання запису.
* `isSparkling: Boolean` - чи була вода газованою.
* `temperature: Float` - температура води.

## Екрани та Навігаційна структура

Для застосунку створено 3 базових екрани. Навігація між основними розділами здійснюється через нижню панель (`NavigationBar`), а перехід до деталей - через внутрішній стек (`NavHost`).

### Перелік екранів
1. **`VitaminListScreen` (Екран Home):** Відображає список доступних вітамінів у вигляді карток. Побудований на базі компонента `LazyColumn`. Дані завантажуються зі статичного об'єкта `MockData`.
2. **`VitaminDetailScreen` (Екран деталей):** Відкривається при кліку на картку вітаміну. Приймає `vitaminId` як аргумент маршруту, знаходить відповідний об'єкт і виводить розширену інформацію (дозування, важливість). Містить кнопку повернення.
3. **`WaterScreen` (Додатковий екран):** Екран для трекінгу випитої води. Доступний через другу вкладку нижнього меню.
