# Звіт з лабораторної роботи №4
**Тема:** Біометрична автентифікація та безпека мобільного застосунку

---

## Мета роботи
Впровадити в застосунок "Water and Vitamin Tracker" надійний механізм біометричної автентифікації. Забезпечити захист доступу до програми через перевірку відбитка пальця або сканування обличчя, обробити всі можливі стани роботи апаратних датчиків та покрити створену логіку безпеки автоматизованими тестами.

## 1. Керування біометрією (Dependency Inversion)

Для чистоти архітектури та можливості легкого тестування логіку автентифікації винесено в абстрактний інтерфейс `AppBiometricManager`. 
Створено дві його імплементації:
1. **`RealBiometricManager`** - використовує офіційну бібліотеку `androidx.biometric` для роботи з реальними датчиками пристрою.
2. **`MockBiometricManager`** - заглушка для Unit-тестування, що дозволяє імітувати поведінку датчика без наявності фізичного апаратного забезпечення.

### Стани автентифікації (`AuthState`)
Процес розпізнавання є асинхронним, тому стан зберігається централізовано у `StateFlow<AuthState>`. Доступні стани:
* `Idle` - очікування дій користувача.
* `Authenticating` - викликано системний діалог біометрії.
* `Success` - підтвердження успішне.
* `Failed` - відбиток не розпізнано або дію скасовано.
* `Unavailable` - на пристрої відсутній датчик або не додано жодного відбитка.

Збереження налаштувань користувача (увімкнення біометрії та час таймауту) реалізовано локально через `SharedPreferences` (`security_prefs`).

## 2. Інтеграція з UI та Нові екрани

Біометричний захист глибоко інтегровано в життєвий цикл застосунку через `LocalLifecycleOwner`. Додано 3 нові екрани/сценарії:

### Екран входу (`LoginScreen`)
* Стартовий екран застосунку. Користувач може увійти за допомогою стандартної форми (логін/пароль) або натиснути кнопку **"Login with Biometrics"**.
* Кнопка біометрії відображається лише якщо користувач увімкнув цю опцію в налаштуваннях. Вбудована обробка помилок (наприклад, виведення "Authentication failed" при скасуванні).

### Екран профілю та налаштувань безпеки (`ProfileScreen`)
* Відображає тип підтримуваного датчика (`FACE_ID`, `FINGERPRINT` або `NONE`).
* Дозволяє увімкнути/вимкнути біометричний вхід за допомогою тумблера (`Switch`).
* Містить налаштування **"Lock app in background after"** (0s, 5s, 15s) у вигляді `FilterChip`, де користувач обирає час таймауту перед блокуванням програми.

### Екран блокування (`LockScreen` та Lifecycle)
* Застосунок відстежує переходи в бекграунд (подія `ON_PAUSE`) та повернення (подія `ON_RESUME`).
* Якщо програма пробула згорнутою довше, ніж вказано в налаштуваннях таймауту, автоматично підіймається прапорець `isAppLocked`, і користувач бачить екран `LockScreen`.
* Зняти блокування можна лише після успішного проходження перевірки (`AuthState.Success`).

## 3. Автоматизоване тестування

Логіку безпеки покрито **12 Unit-тестами** у файлі `BiometricManagerTest.kt`. Для тестів використовується створений раніше `MockBiometricManager`, що гарантує їхнє швидке виконання без прив'язки до Android-емулятора.

**Що перевірено автоматично:**
* Повернення правильного типу датчика (`testCheckAvailabilityReturnsFingerprintWhenAvailable`).
* Перехід у стан `Success` при успішній автентифікації.
* Коректна обробка скасування сканування (стан `Failed`).
* Ізоляція станів налаштувань (збереження та читання прапорця увімкнення біометрії `isEnabledByUser`).
* Скидання стану до `Idle` після завершення перевірок.
