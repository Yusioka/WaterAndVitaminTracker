# Звіт з лабораторної роботи №3
**Тема:** Real-time взаємодія через WebSocket та автоматизоване тестування

---

## Мета роботи
Реалізувати безперервний двосторонній канал зв'язку (WebSocket) для миттєвого отримання даних від сервера. Забезпечити реактивне оновлення інтерфейсу без участі користувача та покрити критичну логіку керування станами з'єднання автоматизованими Unit-тестами.

---

## 1. WebSocket-з'єднання (`MockSocketManager`)

Для керування постійним з'єднанням створено клас `MockSocketManager`, який імітує роботу реального сокет-клієнта з використанням `Kotlin Coroutines`.

### Керування станами (State Machine)
Система чітко відстежує стан підключення через `StateFlow<SocketState>`. Можливі стани:
`Disconnected` ➔ `Connecting` ➔ `Connected` ➔ `Reconnecting`.

### Життєвий цикл (Lifecycle Integration)
Підключення та відключення сокета жорстко прив'язані до життєвого циклу застосунку. У `MainActivity` використано `DisposableEffect` та `LifecycleEventObserver`:
* **`ON_START`** — автоматичне відкриття з'єднання (`connectWs`).
* **`ON_STOP`** — коректне закриття з'єднання (`disconnectWs`), коли застосунок згортається, щоб не витрачати батарею.

### Парсинг даних
Усі вхідні повідомлення надходять у форматі JSON. Для їхньої десеріалізації у типізовану модель `WsMessage` (містить `type`, `text` та `timestamp`) використовується бібліотека **Gson**.

---

## 2. Реактивний UI та Нові екрани

До застосунку додано два нових екрани, доступ до яких налаштовано через оновлену нижню панель навігації (`NavigationBar`).

### Екран "Live Stats" (`StatsScreen`)
Цей екран повністю побудований на реакції на події від сервера:
1. **Статус з'єднання:** Динамічно відображає поточний стан (наприклад, `Connected` або `Reconnecting`).
2. **Жива стрічка повідомлень:** `LazyColumn` автоматично поповнюється новими нагадуваннями ("Time to drink water!"), які надходять від сокета кожні 4 секунди. Екран не потребує "pull-to-refresh".
3. **Обробка розривів:** Додано кнопку "Simulate Connection Drop", яка примусово розриває з'єднання та тестує логіку автоматичного перепідключення (Retry logic).

### Екран "Profile" (`ProfileScreen`)
Додатковий екран, що відображає статистику користувача та його денну ціль щодо споживання води (Daily Goal).

## 3. Автоматизоване тестування

Критичну логіку мережевого менеджера покрито **15 Unit-тестами** (файл `SocketManagerTest.kt`). Для тестування асинхронного коду на базі корутин використано `kotlinx-coroutines-test`.

**Особливості тестування:**
* **Віртуальний час:** Використання `StandardTestDispatcher` та функції `advanceTimeBy()` дозволяє миттєво тестувати затримки (наприклад, очікування перепідключення у 2-5 секунд), не чекаючи цього часу в реальності.
* **Перевірка потоків:** Тести валідують, що повідомлення (`WsMessage`) успішно парсяться з JSON, додаються у `StateFlow` та не надходять після виклику `disconnect()`.
* **Перевірка перепідключення:** Тест `testReconnectEventuallyChangesToConnected` доводить, що система здатна самостійно відновити зв'язок після обриву.
